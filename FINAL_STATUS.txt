🔧 HOW IT WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

EMBEDDINGS (lightweight_embeddings.py):
  1. Tokenize text into words
  2. Calculate TF (term frequency)
  3. Hash words to array indices
  4. Accumulate weights across 384 dimensions
  5. Normalize vector (L2)
  Result: 384-dim vectors like sentence-transformers!

NLP (lightweight_nlp.py):
  Emotion:  Dictionary lookup + scoring
  Sentiment: Keyword presence + intensifier weights
  Intent:   Regex patterns + accumulation
  Result: Fast, deterministic, no hallucinations!

ML (lightweight_ml.py):
  Uses numpy/scipy only (no sklearn needed)
  StandardScaler: mean/std calculation
  PCA: SVD decomposition
  KMeans: standard algorithm
  Result: Mathematically equivalent to sklearn!

IMPORT REDIRECT (lightweight_redirect.py):
  Patches sys.meta_path when imported
  Intercepts all imports of heavy libraries
  Transparently redirects to lightweight versions
  Result: Existing code works unchanged!

Q: Will existing code break?
A: No. Import redirector patches all imports automatically.

Q: Why is accuracy lower?
A: Lightweight models use heuristics instead of neural networks.
   Trade-off: 5-15% accuracy loss for 97% size reduction.

Q: Can I revert?
A: Yes. Just use requirements.txt and update render.yaml.

Q: What if something breaks?
A: Check Render logs and /health endpoint first.

Q: Can I use heavy models if needed?
A: Yes. Set env vars like VENNELA_ENABLE_MINILM=true.

Q: Does this work with GPU?
A: Lightweight mode doesn't need GPU.

Q: Can I fine-tune?
A: Not lightweight models (no gradients). Use heavy models.

Q: Performance characteristics?
A: See LIGHTWEIGHT_DEPLOYMENT.md for detailed benchmarks.
